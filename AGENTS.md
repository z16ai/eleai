# 开发原则

## 核心原则

当用户提出以下需求时：
- 修改或更新某个功能
- 修复某个功能的 bug

必须遵循：
1. **仅修改涉及的功能代码**，不改动其他页面的内容
2. **涉及通用组件修改时**，需先告知用户修改影响范围，征得同意后再操作

---

## 数据库信息

- **项目**: Supabase (ddxcnhidzlzlatzeibon)
- **现有表**:
  - `user_profiles` - 用户资料
  - `image_generations` - 图片生成记录
  - `user_points` - 用户积分 (默认 88 点)
- **后续开发**: 涉及数据功能时，先查询表结构再调整代码
