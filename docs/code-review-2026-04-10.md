# 代码审查报告：eleAI Studio

**审查日期**: 2026-04-10

---

## 一、安全问题（高优先级）

### 1. API 缺少用户认证检查
- **位置**: 所有 `/api/` 路由均未验证用户身份
- **问题**: 任何人都可以调用图片/视频/音频生成 API，消耗服务器资源
- **建议**: 在 API 路由中添加 Supabase 认证检查

### 2. 敏感 API Key 暴露风险
- **位置**: `src/lib/supabase/server.ts:34`
- **问题**: `SUPABASE_SERVICE_ROLE_KEY` 被用于客户端 cookie 操作，可能导致服务端权限泄露
- **建议**: 移除 `createServiceClient` 中的 cookie 操作，或仅在需要严格权限的场景使用

### 3. 硬编码的 App ID
- **位置**: `src/app/api/audio/tts/route.ts:19`
- **问题**: `VOLCENGINE_APP_ID` 使用硬编码默认值 `'BigTTS2000000688695749538'`
- **建议**: 移除默认值，强制要求环境变量配置

---

## 二、代码质量问题（中等优先级）

### 4. 数据存储使用文件系统（不适用于生产环境）
- **位置**: 所有 `list/save` API
- **问题**: 使用本地 JSON 文件存储数据，部署到 Vercel/Netlify 会导致数据丢失
- **建议**: 迁移到 Supabase 数据库或使用 R2 进行持久化存储

### 5. 重复的布局代码
- **位置**: `src/app/*/layout.tsx` (image, video, audio, plaza)
- **问题**: 每个页面都重复定义 Sidebar 和 Footer
- **建议**: 考虑提取公共的 WorkspaceLayout 组件

### 6. 未使用的 Web3 相关代码
- **位置**: `src/components/auth/Web3ConnectButton.tsx`, `UserMenu.tsx`
- **问题**: Web3 登录功能已实现但未在 UI 中完整集成
- **建议**: 完成 Web3 集成或移除未使用的代码

### 7. 类型安全问题
- **位置**: 多个 API 路由使用 `as any` 类型断言
- **位置**: `src/app/api/generate-image/route.ts:145,157,203`
- **建议**: 定义完整的请求/响应类型接口

### 8. API 错误处理不一致
- **位置**: 多个 API 路由
- **问题**: 部分返回 500，部分返回 400，错误信息格式不统一
- **建议**: 统一错误响应格式

## 三、前端/UI 问题（中等优先级）

### 9. 静态图片路径硬编码

- **建议**: 创建 README.md，包含安装、运行、环境变量配置说明

### 11. 缺少 .env.local 示例文件
- **问题**: 没有 `.env.local.example` 文件供开发者参考
- **建议**: 创建示例文件列出所有必需的环境变量

### 12. 空文档
- **位置**: `docs/tts.md` 文件为空
- **建议**: 删除或补充 TTS 相关文档
